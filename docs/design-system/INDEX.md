## The token layer ‚Äî 1:1, no drift

Every Imladris token (`tokens/colors.css`, `typography.css`, `spacing.css`, `effects.css`,
`fonts.css`, `base.css`) was diffed against the theme's `theme.json` and matches exactly: the full
palette (parchment / ink / green / river / gold / twilight + leaf¬∑amber¬∑rust¬∑slate), the spacing
scale, the four font families + type scale, leadings, trackings, type roles, radii, border widths,
shadows, easings and durations. The only DS token-CSS not represented as `theme.json` presets are
alias/util vars (`--gutter-*`, `--focus-*`, the `--surface-*`/`--text-*` semantic aliases), which the
theme's `style.css` already provides via the `--wp--custom--*` / `--wp--preset--*` cascade. A token
"refresh" therefore changes nothing. The faithful token mirror is staged at
`../../.design-pull/imladris-design-system/tokens/`.

`styles.css` in the project is **imports-only** (it `@import`s the six token files), so there is no
component CSS to refresh either ‚Äî the theme's `style.css` is a superset.

## 2026-06-21 ‚Äî header cache-bust (ship the fidelity pass): `Version` 0.3.8 ‚Üí 0.3.9

The SiteHeader/SiteFooter passes above iterated `style.css` repeatedly while the theme `Version`
stayed at `0.3.8`. Since the child stylesheet is cache-busted by that header
(`wp_enqueue_style('hperkins-tokens', ‚Ä¶, wp_get_theme()->get('Version'))` in `functions.php`),
`style.css?ver=0.3.8` became a **moving target** ‚Äî multiple distinct CSS payloads under one cache
key. Clients (and CDN/browser caches) that had cached an earlier `0.3.8` kept rendering the
pre-fix masthead: at ‚â§781px the brand mark stayed visible, the action cluster was hidden, and the
nav wrapped to a **second row** (`flex-wrap: wrap !important` ‚Üí header ~105px), so the mobile header
no longer matched desktop. The desktop in-nav search field (`.hp-nav-search`) likewise still showed.

The **rules were already correct** ‚Äî no CSS change was needed. Bumped `Version` to `0.3.9` (mirrored
in `readme.txt`) so the finalized header CSS ships under a fresh key. Verified end-to-end against the
live site after the bump (real deployed CSS, `?ver=0.3.9`, no injection): single nowrap 61px bar
`[wordmark] ‚Ä¶ [Subscribe] [‚ò∞]`, mark + in-bar search dropped, 44px hamburger, **no overflow at
1440 / 375 / 320**; desktop restores `[mark][wordmark] [inline nav] [search] [Subscribe]` with the
in-nav search field correctly hidden. Lesson: **bump `Version` on every `style.css` edit, not once per
work session** ‚Äî the cache key is content-addressed by that string alone.
