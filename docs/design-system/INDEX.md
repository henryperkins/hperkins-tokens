## The token layer — 1:1, no drift

Every Imladris token (`tokens/colors.css`, `typography.css`, `spacing.css`, `effects.css`,
`fonts.css`, `base.css`) was diffed against the theme's `theme.json` and matches exactly: the full
palette (parchment / ink / green / river / gold / twilight + leaf·amber·rust·slate), the spacing
scale, the four font families + type scale, leadings, trackings, type roles, radii, border widths,
shadows, easings and durations. The only DS token-CSS not represented as `theme.json` presets are
alias/util vars (`--gutter-*`, `--focus-*`, the `--surface-*`/`--text-*` semantic aliases), which the
theme's `style.css` already provides via the `--wp--custom--*` / `--wp--preset--*` cascade. A token
"refresh" therefore changes nothing. The faithful token mirror is staged at
`../../.design-pull/imladris-design-system/tokens/`.

`styles.css` in the project is **imports-only** (it `@import`s the six token files), so there is no
component CSS to refresh either — the theme's `style.css` is a superset.

## Gap report — vendored vs. available-in-project

**Vendored locally** (this folder + the staging mirror):
- `README.md`, this `INDEX.md`
- token mirror — `../../.design-pull/imladris-design-system/tokens/*.css`
- `../../.design-pull/imladris-design-system/styles.css` (the imports-only entry point)

**Not vendored — still only in the project, re-pullable on request:**
- The page design sources `templates/{contact,ai-enablement,work-index}/*.dc.html` — their *designs*
  were implemented as the theme templates/patterns listed above; the raw canvas `.dc.html` was read
  during the pull but not mirrored byte-for-byte.
- Per-component source `components/<group>/<Name>.jsx` (these are *real, editable* source here, not
  just re-export stubs), the `.d.ts` type contracts, and the `.prompt.md` design docs (19 components).
- Guideline specimen cards — `guidelines/*.card.html` (13 cards: brand imagery/logo/three-rings/voice/
  wapuu, colors brand/neutrals/semantic, radii-shadows, spacing-scale, type display/body/labels-mono).
- The compiled `_ds_bundle.js` / `_ds_bundle.css` — **likely exceeds the 256 KiB `get_file` cap**;
  grab from the project's own export if the full bytes are needed.
- The blog `ui_kit` parts (`HomeView.jsx`, `ReaderView.jsx`, `CoverPlate.jsx`, `PostCard.jsx`, …) —
  their *designs* were translated into `home.html`/`single.html`; the React source was not vendored.
- Brand assets (`assets/imagery/*`, `wapuu-*`, `ring-badge-*`) — already present in the theme's
  `assets/img/` and used by the patterns.

To bring the full component/guideline archive local, re-run **`/design-pull`** against the URL above
and ask to vendor components + guidelines, or fetch specific files. The token + CSS layers round-trip
losslessly; the React component layer is reference-only for this block-theme target.
